2. Introduction

The E-Bike Management System is designed to address the growing demand for sustainable urban mobility. By automating tasks like user registration, bike rental and return, and station management, the system optimizes operational efficiency and user experience. It aligns with smart city initiatives by offering eco-friendly transportation solutions that reduce traffic congestion and promote environmental sustainability.

This project employs data structures like arrays, vectors, and graphs, along with algorithms like Dijkstra’s for shortest path calculation. It focuses on real-world problem-solving through efficient resource management, demonstrating the practical applications of design and analysis of algorithms.

3. Problem Definition

3.1 Domain

The E-Bike Management System is rooted in urban transportation management, addressing challenges like:

  • Inefficient bike utilization.
  • Lack of automated user and station management.
  • Difficulties in tracking bike availability across docking stations.

The system automates user registration, invoicing, and bike tracking, enhancing operational efficiency. It also provides tools for administrators to manage stations dynamically and generate daily reports.

3.2 Module Description

I am working on the creation of linear search in which:

  • User Management: Handles registration, login, and authentication of users. Registers a new user by prompting for a username and password. Checks for duplicate usernames to ensure uniqueness. Adds the user to the users vector.
  • Bike Rental/Return: Allows users to rent and return bikes with real-time updates. Updates the available bikes field of the station. Adds a rental transaction to the user's invoices.

Algorithm: Validation checks for availability and station existence.

5. Functionality Analysis

1. User Registration

Workflow:

  1. Collect username and password.
  2. Validate against existing users.
  3. Store user details in a vector.

Efficiency Analysis:

  • Time Complexity: O(n) for duplicate checks.
  • Space Complexity: O(n) for storing user data.

2. Bike Rental and Return

Workflow:

  1. Verify station ID and bike availability.
  2. Update bike count in the array.
  3. Generate an invoice for the user.

Efficiency Analysis:

  • Time Complexity: O(1) for updates.
  • Space Complexity: O(1).
